| Aspect | Detail |
|---|---|
| **Stack** | PyPDF2, spaCy, NLTK, TF-IDF, Sentence-Transformers |
| **Scale** | Tested on a 100-resume batch |
| **Performance** | Sub-second inference per resume |
| **Security** | Local processing pipeline |
| **Impact** | Reduced manual screening effort by an estimated 70% |
| **Repository** | [github.com/Anshika-hey/Resume-parser_final](https://github.com/Anshika-hey/Resume-parser_final) |

Pipeline: PDF ingestion (PyPDF2) → spaCy NER for entity extraction → NLTK preprocessing → TF-IDF vectorization → cosine similarity scoring. Modular, plug-in model architecture extended to handle PDF, DOCX, and plain-text inputs without pipeline changes.

</details>
